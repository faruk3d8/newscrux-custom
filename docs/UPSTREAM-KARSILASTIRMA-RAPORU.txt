1. YÖNETİCİ ÖZETİ
================================================================================

Orijinal Newscrux, RSS kaynaklarından yapay zekâ haberlerini toplayan, OpenRouter
LLM ile özetleyen ve Pushover üzerinden mobil bildirim gönderen tek süreçli bir
Node.js/TypeScript uygulamasıdır. Zamanlama POLL_INTERVAL_MINUTES ile basit
interval döngüsüne dayanır; kullanıcı etkileşimi yoktur.

Özel fork, aynı çekirdek pipeline'ı korurken şu ana eksenlerde genişletilmiştir:

  A) Bildirim kanalı     : Pushover → Telegram Bot API (HTML mesajlar)
  B) Operasyon           : Telegram bot komutları, duraklat/devam, manuel poll
  C) Zamanlama           : Interval yerine Europe/Istanbul saat bazlı (12:00, 20:00)
  D) 3D AI katmanı       : Ayrı RSS/GitHub/arXiv kaynakları, ayrı kuyruk, aylık bütçe
  E) Maliyet izleme      : Token JSONL log, OpenRouter kredi uyarıları, 3D bütçe guard
  F) Raspberry Pi / prod : systemd user unit, linger, dosya logu, deploy script
  G) Güvenlik            : SSRF korumalı HTTP fetch (url-security.ts)
  H) Performans          : Feed profilleri (minimal/full), fast poll, eşzamanlılık
  I) Bot i18n            : TR/EN arayüz (makale özet dili ayrı — i18n.ts)

npm bağımlılıkları değişmemiştir (4 runtime paket). Tüm farklar mimari ve kod
seviyesindedir.

    → setInterval(poll, POLL_INTERVAL_MINUTES)

6. BİLDİRİM: PUSHOVER → TELEGRAM
================================================================================

Orijinal (pushover.ts):
  - Pushover REST API (https://api.pushover.net/1/messages.json)
  - HTML mesaj gövdesi, title düz metin
  - PUSHOVER_USER_KEY + PUSHOVER_APP_TOKEN

Fork (telegram.ts):
  - Telegram Bot API sendMessage
  - HTML parse_mode, escapeHtml()
  - Akıllı truncation (cümle sınırında kesme) — Pushover'dan taşınmış
  - TELEGRAM_BOT_TOKEN + TELEGRAM_CHAT_ID
  - sendChatMessage() — bot komut yanıtları için
  - sendArticleNotification() — makale bildirimleri

Davranış eşdeğeri: Her iki kanal da yapılandırılmış özeti kullanıcıya iletir;
Telegram ek olarak interaktif bot ve grup/kanal mesajlaşması sağlar.

7. TELEGRAM BOT KOMUTLARI
================================================================================

Yetkilendirme: Yalnızca TELEGRAM_CHAT_ID ile eşleşen chat.

Genel komutlar (telegram-commands.config.ts + bot-i18n menü):
  /help, /commands     Komut listesi (locale'e göre TR/EN)
  /status              Son poll, kredi, bütçe, build bilgisi
  /pause, /resume      Zamanlanmış poll'ları duraklat/devam
  /pollnow             Manuel poll (cooldown + kilit; /poll alias)
  /langtr, /langen     Bot arayüz dili (control-state.botLocale)

Telegram menü (BotFather setMyCommands — getTelegramMenuCommands):
  Sabit: commands, status, pollnow
  Dinamik (duruma göre tek satır):
    - Dil: langtr veya langen (mevcut locale'in tersi)
    - Zamanlama: pause veya resume (paused durumuna göre)
    - 3D: 3dainewsopen veya 3dainewsclose (threeDNewsEnabled'a göre)
  Dil değişince registerTelegramBotCommands() menüyü yeniler.

3D komutlar (threed.config.ts):
  /3dainews            3D katman durumu
  /3dainewsopen        3D zamanlanmış poll'ları aç
  /3dainewsclose       3D zamanlanmış poll'ları kapat

Not (pollfast): Kod tabanında fastMode hâlâ desteklenir (config, poll-coordinator)
  ancak ayrı /pollfast bot komutu kaldırılmıştır; manuel tetikleme /pollnow ile yapılır.

Not (poll3d): threed.config.ts içinde poll3d tanımı kalabilir; telegram-bot.ts'te
  handler yok. scripts/diagnose-telegram.sh ve deploy-to-pi.sh hâlâ /poll3d arar — güncel
  kodla uyumsuz (Bölüm 19).

Rate limit: COMMAND_RATE_LIMIT_PER_MINUTE (varsayılan 10/dk per chat)

Not: Makale özet dili CLI --lang ile seçilir (i18n.ts); bot dili ayrıdır (bot-i18n.ts).

13. DAĞITIM VE SYSTEMD (Raspberry Pi)
================================================================================

Orijinal newscrux.service:
  - WantedBy=multi-user.target (sistem servisi)
  - StandardOutput/Error=journal
  - ProtectSystem=strict
  - --lang=en

Fork newscrux.service:
  - User systemd unit (~/.config/systemd/user/)
  - WantedBy=default.target
  - ExecStartPre: mkdir data
  - StandardOutput/Error=append:~/newscrux/data/service.log
  - loginctl enable-linger — SSH/oturum kapalıyken çalışma
  - --lang=tr (Pi varsayılanı)

install-systemd.sh:
  - Unit kopyala, daemon-reload, enable, restart, linger

deploy-to-pi.sh:
  - rsync (node_modules, dist, .env, data hariç)
  - ssh: npm install, npm run build, systemctl --user restart
